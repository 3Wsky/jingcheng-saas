# 开发日志

## 2026-07-13｜商品展示、采集与小程序详情阶段收尾

### 已完成

- 展示商品规格改为按「颜色」与「版本/配置」独立选择，同一配置的各颜色共用价格。
- 商品分类扩展：电脑支持 `Windows` / `Linux`，平板支持悦享款 / 标准版 / 柔光版，智能穿戴仅展示单一款式。
- 后台「编辑展示商品」支持编辑详情图；官网采集会同步主图、轮播图、颜色图、SKU、参数及详情图。
- 官网来源商品的编辑页新增「重新采集」：重新抓取后回填表单，管理员确认保存才落库。
- 电脑详情图使用华为消费者官网图文页：从商城接口的 `graphicDetailValue.appUrl` 读取电脑介绍页，并采集桌面端正文图片；图片经 API 代理供小程序加载。
- 积分商品支持详情图采集、后台编辑、接口返回和小程序展示；旧积分商品会回退读取关联展示商品的详情图。
- 小程序积分商品详情的富文本图片已限制为容器内等比缩放，避免超出页面宽度。

### 验证

- 管理后台生产构建已通过：`npm run build`。
- 服务端路由与官网采集器已通过 Node 语法检查。
- 以 `HUAWEI MateBook 14 酷睿 Ultra`（`prdId=10086627212823`）实测，能从华为消费者官网图文页采集 30 张电脑详情图。

### 已知限制 / 后续事项

- 官网搜索结果目前仍可能包含历史或非当前主推商品；后续如需，可增加「仅当前在售/搜索可见」过滤。
- 部分历史电脑商品（例如 MateBook GT 14）仍有商城商品/SKU 数据，但没有公开图文详情页地址，因此无法采集详情图。
- 小程序代码位于独立仓库 `shunwei-membership`，需要按其发布流程提交版本。

### 发布

- `jingcheng-saas` 推送 `main` 会自动部署后台和 API 至 `ok.xjshunwei.cn`。
- 本阶段代码以当前 `main` 为准；后续有新需求再继续迭代。
